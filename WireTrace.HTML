Jargon Buster

PCAP & Network Terms

📦 PCAP (Packet Capture)
📄 Definition: A file format used to store network packet data captured from a network interface.
Context: Used by tools like Wireshark, tcpdump, and WireTrace for offline analysis.
Example: A .pcap file containing 10,000 packets captured over 5 minutes.
📨 Packet
📄 Definition: A formatted unit of data carried across a network, containing headers and payload.
Context: Each row in the PCAP viewer represents one packet.
Example: An HTTP GET request is a single packet with TCP, IP, and Ethernet headers.
🔌 Protocol
📄 Definition: A set of rules defining how data is formatted and transmitted (e.g., TCP, UDP, DNS).
Context: Packets are classified by protocol in the Protocol column.
Example: DNS protocol packets handle domain name resolution.
🔢 MAC Address
📄 Definition: A unique 48-bit hardware identifier assigned to a network interface controller.
Context: Found in Ethernet frame headers; used for local network communication.
Example: 00:1A:2B:3C:4D:5E — uniquely identifies a device on the LAN.
🔄 ARP (Address Resolution Protocol)
📄 Definition: Protocol mapping IP addresses to MAC addresses on a local network.
Context: ARP packets reveal which devices are communicating on the LAN.
Example: 'Who has 192.168.1.1? Tell 192.168.1.100' — device asking for the router's MAC.
🌐 DNS (Domain Name System)
📄 Definition: Protocol that resolves human-readable domain names to IP addresses.
Context: DNS queries/responses show which websites or services devices are accessing.
Example: Query: 'What is the IP of google.com?' → Response: '142.250.80.46'
🏠 DHCP (Dynamic Host Configuration Protocol)
📄 Definition: Protocol that automatically assigns IP addresses and network config to devices.
Context: DHCP Discover/Offer/Request/ACK sequences show devices joining the network.
Example: DHCP Offer from 192.168.1.1 assigns IP 192.168.1.50 to a new device.
🔄 TCP (Transmission Control Protocol)
📄 Definition: Connection-oriented protocol providing reliable, ordered data delivery.
Context: Most web traffic uses TCP. SYN/ACK/FIN flags show connection lifecycle.
Example: TCP SYN from port 52341 → port 443 initiates an HTTPS connection.
⚡ UDP (User Datagram Protocol)
📄 Definition: Connectionless protocol offering fast, lightweight data transmission without guaranteed delivery.
Context: Used by DNS, DHCP, streaming, gaming, and VoIP.
Example: DNS queries typically use UDP port 53 for speed.
📢 ICMP (Internet Control Message Protocol)
📄 Definition: Protocol for diagnostic and error messages between network devices.
Context: Ping requests/replies and traceroute use ICMP.
Example: Echo Request (Type 8) from 192.168.1.5 to 8.8.8.8 — testing connectivity.
📡 PPPoE (Point-to-Point Protocol over Ethernet)
📄 Definition: Protocol for encapsulating PPP frames inside Ethernet, commonly used by ISPs.
Context: PADI/PADO/PADR/PADS sequences indicate broadband connection establishment.
Example: PADI (PPPoE Active Discovery Initiation) — router trying to connect to ISP.
🔀 VLAN (Virtual Local Area Network)
📄 Definition: A logical subdivision of a LAN, tagged with an ID in the 802.1Q header.
Context: VLAN IDs in packet captures show network segmentation.
Example: VLAN 100 for data traffic, VLAN 200 for voice traffic.
⏱️ TTL (Time To Live)
📄 Definition: Counter in IP headers decremented at each router hop; packet discarded when it reaches zero.
Context: Helps identify routing loops and estimate distance to remote hosts.
Example: TTL=64 suggests the packet originated from a Linux/Mac device.
⚠️ ARP Spoofing
📄 Definition: An attack where a device sends forged ARP replies to associate its MAC with another device's IP.
Context: WireTrace's Security tab detects this — critical MITM attack indicator.
Example: Device 00:AB:CD:EF:12:34 falsely claims to be 192.168.1.1 (the gateway).
🐧 Linux SLL (Linux cooked-mode capture)
🔧 Definition: A special packet header format used when capturing on Linux "any" interface.
Context: When tcpdump uses -i any, all packets get an SLL header wrapping the real protocols underneath.
Example: Source MAC address is extracted from the SLL header.

Data Interpretation Guide

Reading PCAP Data

🔄 TCP Connection Lifecycle
SYN → Client initiates connection
SYN-ACK → Server acknowledges and accepts
ACK → Client confirms — connection established
FIN → Either side closes connection
RST → Connection reset (abrupt termination)
🔍 What to look for: Multiple RST packets to the same destination suggest a service is unavailable or a firewall is blocking.
🌐 DNS Resolution Flow
Query → Device asks "What is the IP for example.com?"
Response (No error) → Server returns the IP address
Response (Name error) → Domain doesn't exist (NXDOMAIN)
Response (Server failure) → DNS server couldn't process the request
🔍 What to look for: Frequent Server Failure responses indicate DNS server issues. Name errors for internal domains suggest misconfiguration.
🏠 DHCP Handshake (DORA)
Discover → Client broadcasts: "I need an IP address"
Offer → Server responds: "You can have 192.168.1.50"
Request → Client: "I'll take 192.168.1.50"
ACK → Server: "Confirmed, it's yours for 24h"
🔍 What to look for: Missing Offer after Discover suggests DHCP server is down. Multiple Discovers indicate the client can't get an address.
📡 PPPoE Session Setup
PADI → Router initiates ISP discovery
PADO → ISP responds with service offer
PADR → Router requests specific service
PADS → ISP confirms session established
PADT → Either side terminates session
🔍 What to look for: PADI without PADO means the ISP is unreachable. Frequent PADT indicates unstable broadband connection.

Reading Event Logs

📶 WiFi Disconnection Patterns
• Repeated DEAUTH for one MAC → Device-specific issue (driver, distance, interference)
• DEAUTH storms across many MACs → AP overloaded or under deauth attack
• DEAUTH Reason=1 → Unspecified reason
• DEAUTH Reason=3 → Leaving the network (normal)
• DEAUTH Reason=7 → Not authenticated
🔍 Tip: Cross-reference with PCAP data to see if the device's traffic shows issues before disconnection.
🌍 WAN Event Patterns
• PPPoE PADI sequences → Router attempting to re-establish ISP connection
• Failed to create socket → Critical: networking stack failure
• Link Up/Down events → Physical line issues (cable, exchange, weather)
🔍 Tip: Check timestamps of WAN events against reported outage times.
